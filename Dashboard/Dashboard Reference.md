# Dashboard Reference — Compact

Compressed 2026-05-05. Original public-product reference notes were removed for
token discipline after `Dashboard/app.py` became the live dashboard.

## Legal Boundary

Use only broad product-design lessons from public pages. Do not copy QuantPad
branding, exact layout, private workflows, prompts, market-data pipelines, or
screen composition.

## Product Lesson To Keep

The dashboard should feel like a dense workbench, not a landing page:

- dark operational shell
- compact metrics above charts
- evidence-first tabs
- short labels
- chart-first decision flow
- no marketing hero sections

## Current Dashboard Purpose

`Dashboard/app.py` is a research instrument:

- evidence board for thesis status
- L2 workbench for Rithmic/Quantower summary and feature comparison
- prop-firm probability view for synthetic LucidFlex phase-risk results
- runbook for the active order-flow data contract

## Visual Defaults

- dark background
- restrained 6-8px radii
- neutral sans text
- monospace for paths/run IDs
- emerald for positive/surviving candidates
- amber for warnings/median loss
- red/rose for breach/high-risk states
